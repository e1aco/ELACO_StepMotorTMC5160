## 核心概念：你的 PID 在做什么

```
┌─────────────────────────────────────────────────────────────┐
│                    你的 PID 控制环                           │
│                                                             │
│  目标位置 ──→ [PID] ──→ 速度命令 ──→ TMC5160 VMAX          │
│      ↑              ↑           │                           │
│      │              │           ▼                           │
│      └──── 编码器反馈 ←── 电机转动 ←── 电机驱动             │
│            (X_ENC)                                         │
│                                                             │
│  关键: PID 输出的是【速度】，不是位置！                      │
│  PID 根据位置误差，计算出应该以多快的速度转动                 │
└─────────────────────────────────────────────────────────────┘
```

### 一句话总结
**位置误差 → PID 计算 → 速度命令 → TMC5160 驱动电机**

## 代码级解析

### 1. PID 计算核心 (ela_pid.c:104-146)

```c
int Pid_Calc(PID_T *pid, int actual)
{
    // 误差 = 目标位置 - 实际位置 (编码器值)
    pid->error = pid->target - actual;

    // 比例项: Kp × error (Q16定点数)
    p_term = (pid->kp * pid->error) >> 16;

    // 积分项: Ki × Σerror (带限幅)
    pid->integral += pid->error;
    i_term = (pid->ki * pid->integral) >> 16;

    // 微分项: Kd × Δerror
    pid->derivative = pid->error - pid->error_prev;
    d_term = (pid->kd * pid->derivative) >> 16;

    // 输出 = P + I + D (限幅后写入 VMAX)
    return pid->output;
}
```

### 2. PID 控制循环 (elaco_main.c:272-309)

```c
void Motor_PidProc(void)  // 10kHz 调用
{
    for (每个电机) {
        1. actual = 读编码器位置 (X_ENC 寄存器)
        2. velocity = Pid_Calc(pid, actual)  // PID 计算
        3. Tmc5160_SetVelocity(chip, velocity)  // 写 VMAX
        4. 如果 |error| < 50 → 停止 + 发到位反馈
    }
}
```

### 3. Q16 定点数换算

代码用 `>> 16` 做定点数运算：
```
实际浮点值 = 定点整数值 / 65536

当前默认参数:
  Kp = 8192  → 0.125
  Ki = 2048  → 0.03125
  Kd = 4096  → 0.0625
```

### 4. TMC5160 速度控制 (ela_tmc5160.c:376-390)

```c
void Tmc5160_SetVelocity(TMC5160_T *chip, int velocity)
{
    if (velocity >= 0) {
        RAMPMODE = 1;      // 正转模式
        VMAX = velocity;   // 速度值
    } else {
        RAMPMODE = 2;      // 反转模式
        VMAX = -velocity;  // 速度值取正
    }
}
```

## 物理意义计算

### 编码器 vs 微步 的单位关系

```
1 圈 = 4000 编码器计数 = 51,200 微步

换算系数: 51,200 / 4000 = 12.8 微步/编码器计数

换句话说:
  编码器误差 100 = 1280 微步误差 = 约 9° 机械角度
  编码器误差 50  = 640 微步误差  = 约 4.5° 机械角度 (你的到位阈值)
  编码器误差 10  = 128 微步误差  = 约 0.9° 机械角度
```

### PID 输出与实际速度

TMC5160 的 VMAX 单位是 **时钟周期/微步**：
```
实际速度 (微步/秒) = 内部时钟 / VMAX
内部时钟 ≈ 16MHz (带预分频)

示例:
  VMAX = 1000  → 16,000 微步/秒 → 约 18.75 RPM
  VMAX = 2000  → 8,000 微步/秒  → 约 9.375 RPM
  VMAX = 5000  → 3,200 微步/秒  → 约 3.75 RPM
  VMAX = 50000 → 320 微步/秒    → 约 0.375 RPM
```

**注意**: 你的 `out_max = 50000` 对应约 0.375 RPM，这非常慢！
**建议**: 可能需要增大到 2000~5000 范围 (约 4~10 RPM)

## 当前参数的问题分析

### 问题 1: out_max 太小
```c
pid->out_max = 50000;   // 约 0.375 RPM - 太慢！
pid->out_min = -50000;
```
**建议**: 改为 2000~5000 (约 4~10 RPM)

### 问题 2: 到位阈值可能太大
```c
if (pid->error > -50 && pid->error < 50)  // 50 编码器计数 = 4.5°
```
**分析**: 4.5° 的到位精度是否足够？取决于你的应用。

### 问题 3: 积分清零条件太激进
```c
if (pid->error > -10 && pid->error < 10)  // 误差 < 10 时清零积分
```
**问题**: 这会导致积分项几乎不起作用，因为误差稍小就清零了。
**建议**: 改为 5 或更小，或者去掉这个条件。

### 问题 4: Kp 可能太小
```
当前 Kp = 0.125
当误差 = 1000 编码器计数时:
  P 输出 = 0.125 × 1000 = 125 (VMAX 单位)
  对应速度 ≈ 16MHz / 125 ≈ 128,000 微步/秒 ≈ 150 RPM
```
这个速度是合理的，但需要确认是否会导致震荡。

## 调参步骤建议

### 第一步: 先用纯 P 控制

```
1. 设 Ki=0, Kd=0
2. 从小 Kp 开始 (比如 4096 = 0.0625)
3. 发送一个目标位置，观察:
   - 电机是否能跟随？
   - 是否有震荡？
   - 响应速度如何？
4. 逐步增大 Kp 直到出现震荡，然后减半
```

### 第二步: 加入积分项

```
1. 如果有稳态误差 (电机停在目标附近但没到位)
2. 从小 Ki 开始 (比如 512 = 0.0078)
3. 观察稳态误差是否消除
4. 如果有积分饱和 (overshoot)，减小 Ki 或增大 integral_max
```

### 第三步: 加入微分项

```
1. 如果响应太慢或有超调
2. 从小 Kd 开始 (比如 2048 = 0.03125)
3. 观察超调是否减小
4. 如果有高频抖动，减小 Kd
```

### 第四步: 调整限幅参数

```
out_max/out_min: 控制最大速度
integral_max: 控制积分饱和
```

## 根据精度要求的参数计算

### 到位阈值调整

```
目标精度: ±1°
1° = 360° / 360 = 1°
编码器每度: 4000 / 360 ≈ 11.1 编码器计数

所以到位阈值应该改为: ≤11 (当前是 50)
建议值: 8~12 编码器计数
```

### 建议修改 (elaco_main.c:297)

```c
// 当前:
if (pid->error > -50 && pid->error < 50)

// 建议改为:
if (pid->error > -11 && pid->error < 11)  // ±1° 精度
```

## 系统性测试方案

### 测试 1: 纯 P 控制 (基础)

```
目标: 找到合适的 Kp 范围

步骤:
1. 通过 CAN 0x06 设置: Ki=0, Kd=0, out_max=2000
2. 发送 CMD_ABS_POSITION (0x01) 到一个目标位置
3. 观察:
   - 电机是否能动？
   - 是否震荡？
   - 响应速度？
4. 如果震荡 → 减小 Kp
5. 如果不动/太慢 → 增大 Kp

推荐起始 Kp 值:
  Kp = 4096 (0.0625) → 小
  Kp = 8192 (0.125)  → 中 (当前默认)
  Kp = 16384 (0.25)  → 大
```

### 测试 2: 加入积分 (消除稳态误差)

```
目标: 消除稳态误差

步骤:
1. 在测试1的基础上，找到一个不震荡的 Kp
2. 通过 CAN 0x06 设置小 Ki (比如 512 = 0.0078)
3. 观察:
   - 稳态误差是否消除？
   - 是否有超调？
4. 如果超调 → 减小 Ki 或增大 integral_max
```

### 测试 3: 加入微分 (减少超调)

```
目标: 减少超调和震荡

步骤:
1. 在测试2的基础上
2. 通过 CAN 0x06 设置小 Kd (比如 2048 = 0.03125)
3. 观察:
   - 超调是否减小？
   - 是否有高频抖动？
4. 如果抖动 → 减小 Kd
```

### 测试 4: 调整限幅

```
out_max: 控制最大速度
  - 太小: 响应慢
  - 太大: 可能震荡或堵转
  - 建议: 从 2000 开始，逐步增大

integral_max: 控制积分饱和
  - 太小: 积分不起作用
  - 太大: 可能积分饱和导致超调
  - 建议: 从 16384 开始
```

## CAN 调参命令格式

```
发送 ID: 0x1AA55F42
格式:
  byte[0-3] = 参数值 (小端序)
  byte[4]   = 0x06 (CMD_PID_TUNING)
  byte[5]   = 电机号 (0x01 或 0x02)
  byte[6]   = 参数类型:
              0x01 = Kp (Q16定点数)
              0x02 = Ki (Q16定点数)
              0x03 = Kd (Q16定点数)
              0x04 = out_max (直接整数)
              0x05 = out_min (直接整数)
              0x06 = integral_max (直接整数)

反馈 ID: 0x1AA55F43
格式:
  byte[0-3] = 参数值
  byte[4]   = 参数类型
  byte[5]   = 电机号
  byte[6]   = 0x06 (标识为调参反馈)
```

### 调参示例 (Python)

```python
# 设置 Kp = 0.25 (Q16: 0.25 * 65536 = 16384)
kp_q16 = 16384
data = [
    kp_q16 & 0xFF,           # byte[0]
    (kp_q16 >> 8) & 0xFF,    # byte[1]
    (kp_q16 >> 16) & 0xFF,   # byte[2]
    (kp_q16 >> 24) & 0xFF,   # byte[3]
    0x06,                     # byte[4] = CMD_PID_TUNING
    0x01,                     # byte[5] = 电机1
    0x01,                     # byte[6] = PID_PARAM_KP
    0                         # byte[7] = 校验和 (需计算)
]
```
