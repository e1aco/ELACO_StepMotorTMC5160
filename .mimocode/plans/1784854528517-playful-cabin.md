# S7: 芯片状态结构体 + Flash 持久化

## 目标
将 `closed_loop` 和 `mode` 持久化到 STM32 内部 Flash，掉电保持。

## 数据结构

### Flash 存储布局（Sector 4, 0x08010000, 64KB）

```
偏移 0x00: CHIP_CONFIG_T (chip1)  ← 8 bytes
偏移 0x08: CHIP_CONFIG_T (chip2)  ← 8 bytes
偏移 0x10: 校验区 (reserved)
```

### CHIP_CONFIG_T（新增）

```c
typedef struct {
    uint32_t magic;       // 0x454C4143 ("ELAC")
    uint8_t  mode;        // TMC5160_MODE_POSITION / TMC5160_MODE_VELOCITY
    uint8_t  closed_loop; // CLOSED_LOOP_OFF / CLOSED_LOOP_ON
    uint16_t checksum;    // 前 6 字节累加
} CHIP_CONFIG_T;          // 8 bytes, 4-byte aligned
```

### TMC5160_CHIP_T 修改

```c
typedef struct {
    uint8_t chip_number;
    uint8_t mode;
    uint8_t closed_loop;   // 新增
} TMC5160_CHIP_T;
```

## 步骤

### S7-A: 移植 EEPROM 模块
- 从 `Firmware/StepMotorDriver_Keil/ELA_LIB/ela_eeprom.c/h` 复制到 AIFirmware 版
- 适配：头文件 include 路径调整（`elaco_main.h` → 独立 `stdint.h`）
- EEPROM 模块独立于业务逻辑，不依赖 elaco_main.h

### S7-B: 修改 TMC5160_CHIP_T + 新增配置持久化
- `ela_tmc5160_usr.h`：结构体增加 `closed_loop` 字段
- 新增 `CHIP_CONFIG_T` 类型定义（放在 `ela_tmc5160_usr.h` 或单独头文件）
- Flash 地址宏定义：`CHIP_CONFIG_ADDR 0x08010000`

### S7-C: 实现配置加载/保存函数
在 `ela_tmc5160_usr.c` 中新增：

```c
static void ela_tmc5160_load_config(TMC5160_CHIP_T *chip);
static void ela_tmc5160_save_config(TMC5160_CHIP_T *chip);
```

- `load_config`：从 Flash 读取 CHIP_CONFIG_T，校验 magic + checksum
  - 有效 → 赋值 `chip->mode` 和 `chip->closed_loop`
  - 无效（首次上电）→ 使用默认值 `TMC5160_MODE_POSITION` + `CLOSED_LOOP_OFF`
- `save_config`：将 chip 的 mode/closed_loop 写入 Flash
  - 先比对 Flash 中现有内容，相同则跳过（减少擦写）
  - 不同 → 擦除 Sector 4 → 写入两个 chip 的配置

### S7-D: 修改初始化流程
`ela_tmc5160_init()` 修改：
1. 设置 `chip_number`（不变）
2. **调用 `ela_tmc5160_load_config(chip)` 从 Flash 加载 mode 和 closed_loop**
3. 用加载的 mode 调用 `tmc5160_drv_set_mode()`
4. 使能电机、清除错误、配置寄存器（不变）

### S7-E: 修改闭环控制模块
`ela_closed_loop.c` 修改：
- 删除 `static uint8_t g_cl_mode[2]`
- 内部新增 `get_chip()` 映射 motor ID → `TMC5160_CHIP_T*`（include `ela_tmc5160_usr.h`，直接用全局变量 `g_tmc5160_chip1_st`/`g_tmc5160_chip2_st`）
- `ela_closed_loop_enable()`：`get_chip(motor)->closed_loop = CLOSED_LOOP_ON`
- `ela_closed_loop_disable()`：`get_chip(motor)->closed_loop = CLOSED_LOOP_OFF`
- `ela_closed_loop_get_mode()`：`return get_chip(motor)->closed_loop`
- `ela_closed_loop_tick()`：读取 `chip->closed_loop` 替代 `g_cl_mode[idx]`

**函数签名不变**（保持 `uint8_t motor` 参数），CAN 命令处理接口无需改动。

### S7-F: 修改 CAN 命令处理
`ela_can_usr.c` 中 0x07/0x08 命令仅增加脏标记：
```c
case CMD_CL_ENABLE:
    ela_closed_loop_enable(cmd.motor);
    ela_tmc5160_set_dirty();   // 新增：置脏标记
    ela_can_usr_send_motion_feedback(...);
    break;
```

### S7-G: 主循环添加 Flash 写入
`elaco_main.c` 主循环：
```c
while (1) {
    ela_can_usr_process();
    ela_tmc5160_flush_config();  // 新增：检测脏标记，写入 Flash
}
```

`ela_tmc5160_flush_config()` 逻辑：
- 检查全局脏标记 `g_chip_dirty`
- 有脏 → 调用 `ela_tmc5160_save_config()` 保存两个 chip
- 清除脏标记

### S7-H: Keil 工程同步
- `ela_eeprom.c` 添加到 `.uvprojx` 的 ELA_LIB 组

## 验证
1. 编译通过（Keil）
2. 上电后读取 Flash，首次应为默认值（mode=POSITION, closed_loop=OFF）
3. 通过 CAN 0x07 使能闭环 → 断电 → 重启 → 闭环应自动恢复
4. 通过 CAN 切换模式 → 断电 → 重启 → 模式应保持
5. 连续快速发送 CAN 命令 → Flash 不应每次都擦写（内容相同则跳过）

## 关键决策
- [2026-07-24] Flash 使用 Sector 4 (0x08010000, 64KB)，与 Firmware 版 EEPROM 分配一致
- [2026-07-24] 脏标记 + 内容比对，减少 Flash 磨损
- [2026-07-24] 掉电不丢失，闭环状态持久化
