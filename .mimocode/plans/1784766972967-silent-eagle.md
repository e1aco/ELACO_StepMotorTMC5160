# TMC5160 编码器接口分析

## 问题现象

- XACTUAL = 51200（正确，一圈）
- X_ENC = -51230（反向，且有30的偏移）

## 根因分析

### 1. ENC_CONST 值是正确的

根据 TMC5160 数据手册第 108-109 页：

**ENC_CONST 计算公式：**
```
Factor = FSC × USC / encoder resolution
       = 200 × 256 / 4000
       = 12.8
```

**当前 ENC_CONST = 0x000CCCCD 的含义：**
- 这是 16.16 定点数（二进制模式）
- 整数部分：0x000C = 12
- 小数部分：0xCCCCD / 65536 ≈ 0.8
- 实际值：12.8 ✓

数据手册明确说 4000 CPR 编码器对应 factor 12.8，所以 ENC_CONST 配置正确。

### 2. 编码器方向反了

数据手册第 108 页明确说明：

> "Negating the sign of ENC_CONST allows inversion of the counting direction to match motor and encoder direction."

**反转编码器方向的方法是取反 ENC_CONST，不是改 ENCMODE！**

### 3. ENCMODE 的 pol_A/pol_B 不控制计数方向

ENCMODE 寄存器定义（数据手册第 47 页）：
- bit0: pol_A — N 通道事件所需的 A 相极性
- bit1: pol_B — N 通道事件所需的 B 相极性

这两个位**只影响 N 通道事件检测**，不影响编码器计数方向。这就是为什么改 ENCMODE=0x01 后电机不转的原因（可能影响了其他功能）。

## 解决方案

### 方案 A：取反 ENC_CONST（推荐）

```c
// 当前值（正向计数）
#define ENC_CONST_POSITIVE  0x000CCCCD

// 反向值（负向计数）
#define ENC_CONST_NEGATIVE  0xFFF33333  // 0x000CCCCD 的二进制补码
```

修改 `ela_tmc5160_config_encoder()`：
```c
ela_tmc5160_write_reg(chip, REG_ENC_CONST, 0xFFF33333);
```

### 方案 B：软件取反 X_ENC

在读取编码器位置后手动取反：
```c
int32_t x_enc = -ela_tmc5160_get_encoder_position(chip);
```

## 关于 30 的偏移

增量编码器上电时没有机械零点，X_ENC 的初始值取决于上电瞬间编码器的位置。这是正常现象。

如果需要精确对零，可以：
1. 使用编码器的 N 通道（索引信号）进行零点校准
2. 或者在软件中记录上电时的偏移量并补偿

## 需要修改的文件

- `ELA_LIB/ela_tmc5160_usr.c` → `ela_tmc5160_config_encoder()`
  - 将 ENC_CONST 从 0x000CCCCD 改为 0xFFF33333
  - 或者保持原值，在 motor_ctrl 层取反 X_ENC

## 验证方法

修改后烧录，发 CAN 指令，检查：
1. XACTUAL 和 X_ENC 应该都是正值
2. X_ENC 应该接近 XACTUAL（偏差应该在几十以内）
