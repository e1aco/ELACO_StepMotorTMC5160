# TMC5160 软件闭环控制方案

## 背景

根据 TMC5160 数据手册第 109 页 "20.3 Closing the Loop"：

> TMC5160 支持通过编码器实现软件闭环：
> - 读取编码器位置 (X_ENC)
> - 与目标位置比较
> - 根据偏差修改目标位置
> - 或者定期用编码器位置更新实际位置

真正的硬件闭环需要 TMC4361，但 TMC5160 可以通过 MCU 软件实现闭环控制。

## 实现方案

### 核心思路

在定时器中断中执行闭环控制：
1. 读取编码器位置 (X_ENC) 作为实际位置
2. 读取目标位置 (XTARGET)
3. 计算偏差 = 目标位置 - 编码器位置
4. 通过 PID 算法计算修正量
5. 将修正量加到目标位置上

### 实现步骤

#### S3-A: PID 控制器模块
- 新增 `ela_pid.c/h`
- 实现增量式 PID 算法
- 支持参数在线调整（Kp, Ki, Kd）
- 输出限幅

#### S3-B: 闭环控制模块
- 新增 `ela_closed_loop.c/h`
- 定时器中断中执行闭环控制周期（2ms）
- 读取 X_ENC 作为反馈
- 调用 PID 计算修正量
- 更新 XTARGET

#### S3-C: 闭环模式切换
- 修改 `ela_motor_ctrl` 支持开环/闭环模式切换
- 闭环模式下：PID 控制目标位置
- 开环模式下：直接设置目标位置（当前行为）

#### S3-D: 集成测试
- 测试闭环定位精度
- 测试抗干扰能力（手动施加阻力）
- 调试 PID 参数

### 关键代码片段

```c
// 闭环控制周期函数（定时器中断调用，2ms 周期）
void ela_closed_loop_tick(void)
{
    // 1. 读取编码器位置（实际位置）
    int32_t actual = ela_tmc5160_get_encoder_position(chip);
    
    // 2. 读取目标位置
    int32_t target = ela_tmc5160_get_position(chip);
    
    // 3. PID 计算修正量
    int32_t correction = ela_pid_calculate(&g_pid_st, target, actual);
    
    // 4. 更新目标位置
    int32_t new_target = target + correction;
    ela_tmc5160_move_to(chip, new_target);
}
```

### 控制周期选择

**推荐：2ms (500Hz)**

依据：
- 步进电机机械时间常数 ≈ 20ms
- 控制周期 ≤ 机械时间常数 / 10 = 2ms
- CPU 负载适中（每次中断约 100us）
- 与 CAN 通信周期匹配

### 依赖的硬件资源

- 定时器（如 TIM7）用于闭环控制周期
- 编码器接口（已配置）
- SPI 接口（已配置）

### 验证方法

1. **定位精度测试**：发送目标位置，比较 X_ENC 与 XTARGET 的稳态误差
2. **抗干扰测试**：电机运行时手动施加阻力，观察是否能自动修正
3. **响应速度测试**：阶跃响应时间、超调量
