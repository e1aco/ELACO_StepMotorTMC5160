# ELA_LIB 模块化复用计划（v3）

## 核心思路

**模块本身 = 抽象单元**。不建新文件，不改架构，在现有分层规则下：

1. 模块头文件独立（不 include elaco_main.h）
2. drv 区段是唯一接触 HAL 的地方
3. 核心文档(.lnk)维护引脚映射表，移植时查表填 drv

## 目录结构

```
项目根目录/
├── Require/              ← 需求文档、数据手册、引脚映射表
│   ├── *.lnk            ← 核心文档（快捷方式）
│   ├── *.pdf            ← 数据手册
│   └── *.md             ← 其他参考
├── ELA_LIB/              ← 可复用代码模块（与 Require 同级）
│   ├── ela_pid.c/h      ← A级：纯计算，直接复用
│   ├── ela_queue.c/h    ← A级：纯数据结构，直接复用
│   ├── ela_button.c/h   ← B级：小改 drv 区段即可
│   ├── ela_tmc5160.c/h  ← C级：需改引脚配置
│   ├── ela_can.c/h      ← D级：需改协议配置
│   ├── ela_eeprom.c/h   ← D级：需改 Flash 地址
│   └── elaco_main.c/h   ← 项目专属，不复用
└── ...（其他项目文件）
```

新项目初始化时：`/em init` 自动创建 `ELA_LIB/` 目录（如不存在）。

## 架构

```
模块文件 ela_xxx.c 内部:
┌─────────────────────────────────────┐
│  hlp 区段 → 纯计算，零 HAL         │ ← 直接复用
├─────────────────────────────────────┤
│  drv 区段 → 唯一接触 HAL 的地方     │ ← 移植时只改这里
├─────────────────────────────────────┤
│  usr 区段 → 只调用 drv 接口         │ ← 直接复用
└─────────────────────────────────────┘

核心文档 内容:
┌─────────────────────────────────────┐
│  引脚映射表 (Pin Mapping)           │ ← 查表填 drv 区段
│  TMC5160 U1: CS=PD5, DRV_ENN=PG13  │
│  TMC5160 U2: CS=PB0, DRV_ENN=PB1   │
│  ...                                │
└─────────────────────────────────────┘
```

在第十节「函数分层」中新增「模块独立性约束」，并新增「二十、elaco_main.c 职责约束」：

#### 模块独立性约束

1. **头文件独立**: 每个 ela_xxx.h 只 include 自己需要的头文件
   - 允许: stdint.h, stdbool.h, 本模块 drv 需要的 HAL 头文件
   - 禁止: elaco_main.h, 其他 ela_xxx.h

2. **drv 区段自包含**: drv 区段是唯一接触 HAL 的地方
   - hlp/usr 区段禁止 #include stm32f4xx_hal_xxx.h

3. **移植检查清单**: 复制模块后必须检查
   - [ ] 头文件是否独立
   - [ ] drv 区段 HAL handle 是否适配
   - [ ] 引脚定义是否查表填写
   - [ ] 编译通过 + 功能正常

#### elaco_main.c 职责约束

**elaco_main.c 放 HAL 回调 + 主循环 + 初始化，其他什么都不要放。**

允许放:
- 主循环 elaco_main() — 只做调度
- HAL 回调函数本体（HAL_TIM_PeriodElapsedCallback、HAL_CAN_RxFifo0MsgPendingCallback 等）
- 模块初始化调用

禁止放:
- 业务逻辑 → 放对应模块的 usr 区段
- 协议解析 → 放对应通信模块的 usr 区段
- 测试代码 → 放 test/ 目录
- 非 HAL 回调的函数 → 归到各自模块文件

回调函数原则:
```c
/* elaco_main.c — HAL 回调留在这里，内部只分发 */
void HAL_TIM_PeriodElapsedCallback(TIM_HandleTypeDef *htim) {
    if (htim == &htim5) {
        Button_ScanProc();      /* 具体实现在 ela_button.c */
    }
    if (htim == &htim9) {
        Pid_Calc();             /* 具体实现在 ela_pid.c */
    }
}

void HAL_CAN_RxFifo0MsgPendingCallback(CAN_HandleTypeDef *hcan) {
    Can_ReceiveMessage();       /* 具体实现在 ela_can.c */
}
```

行数约束: elaco_main.c ≤ 200 行。

### ela_queue 改造（A级，无 HAL 依赖）

**ela_queue.h 改动:**
1. 加文件头注释（ela_rules 格式）
2. `#ifndef _ELA_QUEUE_H_` → `#ifndef ELA_QUEUE_H`（去掉前导下划线）
3. 加 `#include <stdint.h>`
4. `CAN_LENGTH` → `QUEUE_ELEM_SIZE`（通用化命名）
5. `TRUE/FALSE` → `QUEUE_OK/QUEUE_FULL`（避免宏冲突）
6. `unsigned char` → `uint8_t`
7. `signed char` → `int8_t`
8. `QUEUE_T` 类型注释更新

**ela_queue.c 改动:**
1. 加文件头注释
2. 宏名适配: `CAN_LENGTH` → `QUEUE_ELEM_SIZE`, `TRUE/FALSE` → `QUEUE_OK/QUEUE_FULL`
3. 全局变量: `g_can_queue_st` → `g_queue_st`（通用化）
4. `g_all_can_count` → `g_queue_count`

**改造后 ela_queue.h 预览:**
```c
/****
 * @ 文件: ela_queue.h
 * @ 作者: ELA
 * @ 日期: 2026-07-21
 * @ 版本: 2.0.0
 * @ 说明: 通用循环队列模块，零硬件依赖，可直接复用
 ********/
#ifndef ELA_QUEUE_H
#define ELA_QUEUE_H

#include <stdint.h>

#define QUEUE_SIZE      48
#define QUEUE_ELEM_SIZE 8
#define QUEUE_OK        0
#define QUEUE_FULL      1
#define QUEUE_NULL_PTR  ((void *)0)

typedef struct {
    uint8_t _queue[QUEUE_SIZE][QUEUE_ELEM_SIZE];
    int8_t front;
    int8_t rear;
} QUEUE_T;

extern QUEUE_T g_queue_st;

uint8_t Queue_Insert(QUEUE_T *me, uint8_t *value);
uint8_t Queue_Delete(QUEUE_T *me);
void   *Queue_First(QUEUE_T *me);
uint8_t Queue_IsFull(QUEUE_T *me);
uint8_t Queue_IsEmpty(QUEUE_T *me);
void    Queue_Init(QUEUE_T *me);

#endif
```
