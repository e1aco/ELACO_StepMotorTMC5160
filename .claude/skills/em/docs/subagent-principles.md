# EM-SKILL 子 Agent 调度原则

> 本文档定义 EM-SKILL 中何时、如何使用子 Agent（并行调度）。
> 所有命令文件如需调用子 Agent，遵循本文档约定。

## 核心原则

1. **子 Agent 不是「更快」，是「不串台」** — 每个子 Agent 有独立的上下文，不会因为上下文过长而遗忘细节
2. **只拆无依赖的任务** — 子任务之间必须无数据依赖
3. **失败降级，不阻断** — 任一子 Agent 失败 → 主 Agent 自行处理该路，不阻塞其他路
4. **不做轻量任务** — 单文件修改、简单 bugfix、轻档 new **不启用**子 Agent

## 触发条件

| 场景 | 触发条件 | 最大并行数 | 路由 |
|------|---------|-----------|------|
| 14 项模块审核（mode audit） | 无（默认启用） | 4 | `commands/mode.md` |
| 重档 brainstorm（≥3 子系统） | 子系统数 ≥ 3 | 3 | `commands/new.md` |
| verify 并行：编译+静态分析+代码审查 | deep 档 verify | 3 | `commands/verify.md` |
| 学习模式的独立知识库查询 | 用户明确要求 | 2 | `plugins/learning/commands/learn-kb.md` |
| 其他场景 | 主 Agent 自行判断是否可拆 | ≤ 3 | 对应命令文件 |

## 调度协议

### 主 Agent 职责

1. **拆分任务**：把大任务拆成独立子任务，确保无数据依赖
2. **打包上下文**：每个子任务附带所需的完整上下文（源码、配置文件、相关文档）
3. **派发**：并发通过 Agent 工具派发
4. **收集**：等待所有子 Agent 完成（或超时/失败）
5. **汇总**：整合子 Agent 结果，检测冲突/矛盾
6. **输出**：呈现给用户

### 子 Agent 职责

1. **只做自己的事**：不关心其他子 Agent 的存在
2. **返回结构化结果**：格式见各命令定义
3. **超时处理**：120 秒内未返回 → 标记失败由主 Agent 兜底

## 失败处理策略

| 失败类型 | 处理方式 |
|---------|---------|
| 子 Agent 返回空/报错 | 主 Agent 自己处理该路，备注 `⚠️ Agent 降级（串行）` |
| 部分子 Agent 成功 | 成功的用结果，失败的降级，最终统一输出 |
| 全部失败 | 退化为完整的串行流程 |

## 具体实现示例

### 例 1：14 项审核（mode.md）

```yaml
主任务: /em mode audit ela_xxx
派发:
  - Agent-1: 命名组（4 项）→ 源码给 Agent，只检查命名
  - Agent-2: 注释组（3 项）→ 源码给 Agent，只检查注释
  - Agent-3: 格式组（4 项）→ 源码给 Agent，只检查格式
  - Agent-4: 安全组（3 项）→ 源码给 Agent，只检查安全
汇总: 4 组结果合并为 14/14 表
降级: 某组失败 → 主 Agent 亲自检查该组
```

### 例 2：多子系统 brainstorm（new.md）

```yaml
主任务: brainstorm 3 个子系统技术方案
派发:
  - Agent-1: 子系统A（CAN通信）方案调研
  - Agent-2: 子系统B（PID 控制）方案调研
  - Agent-3: 子系统C（电流检测）方案调研
汇总: 合并方案 → 检测引脚冲突/资源争用 → 输出 brainstorm.md
降级: 某子系统方案失败 → 主 Agent 自行给出兜底方案
```
