# 命令: /em mode (模块管理)

## 功能
管理 ELA_LIB 可复用代码模块。模块加入模块库前必须通过 14 项验证。

## 触发
```
/em mode list              # 列出模块库中可用的模块
/em mode audit <模块名>    # 按 ela_rules.md 审核模块（14项验证）
/em mode add <模块名>      # 将模块加入模块库（需先通过审核）
/em mode install <模块名>  # 从模块库复制到当前项目 ELA_LIB/
```

## 模块库位置

`skills/em/modules/` — 存放审核通过的可复用模块源码。

## 加入模块库流程

```
1. 准备模块文件（ela_xxx.c/h）
   ↓
2. /em mode audit <模块名>  ← 必须 14/14 全通过
   ↓
3. /em mode add <模块名>    ← 审核通过后才能加入
   ↓
4. 模块存入 skills/em/modules/
```

## /em mode list

扫描 `skills/em/modules/` 目录，输出可用模块：

```
📦 模块库

| 模块 | 等级 | 说明 |
|------|------|------|
| ela_queue | A | 通用循环队列，零硬件依赖 |
```

## /em mode audit <模块名>

按 ela_rules.md 第二十一节「代码验证规则」逐项审核。

### 审核模式选择

| 条件 | 模式 |
|------|------|
| 命令带 `--serial` | 强制串行（传统的逐项检查） |
| 默认 | **并行审核**（4 组子 Agent 同时跑） |
| 子 Agent 执行失败 | 自动降级为串行，不阻塞 |

### 并行审核模式（默认，推荐）

14 项审核拆为 4 组**无依赖**的子任务，通过 Agent 工具并发执行：

```
主 Agent: 分配 4 组任务
  │
  ├─ Agent-1（命名组）── 4 项
  │     □ 函数命名 ............. ✅ / ❌
  │     □ 静态函数命名 ......... ✅ / ❌
  │     □ 变量命名 ............. ✅ / ❌
  │     □ 类型命名 ............. ✅ / ❌
  │
  ├─ Agent-2（注释组）── 3 项
  │     □ 文件头注释 ........... ✅ / ❌   (含 @ 依赖 检查)
  │     □ 头文件保护名 ......... ✅ / ❌
  │     □ 函数注释 ............. ✅ / ❌
  │
  ├─ Agent-3（格式组）── 4 项
  │     □ 段标记命名 ........... ✅ / ❌   (模块名_层，不用 module)
  │     □ drv 区段验证 ......... ✅ / ❌
  │     □ 代码格式 ............. ✅ / ❌
  │     □ Magic Number ......... ✅ / ❌
  │
  └─ Agent-4（安全组）── 3 项
        □ 头文件独立性 ......... ✅ / ❌
        □ 宏命名 ............... ✅ / ❌
        □ volatile ............. ✅ / ❌
```

**执行规则**：
1. 主 Agent 把模块的完整源码（.c/.h）分别传给 4 个子 Agent
2. 每个子 Agent 独立运行，互不依赖
3. 所有子 Agent 完成后，主 Agent 汇总输出

**合并仲裁规则**：
- 同一项有多个 Agent 给出结论 → 取**最严格**的那个（任一 Agent 报 ❌ → 该项 ❌）
- 4 组的检查结果取**并集**，不取交集
- 汇总表中每一行标注来源：
  ```
  □ 函数命名 ............. ❌   ← Agent-1
  □ volatile ............. ✅   ← Agent-4
  ```
- 如发现跨组冲突（如 Agent-1 说命名 OK，但 Agent-3 说段标记名不符 → 其实两者指向同一个问题），统一合并为一条违规项

**失败处理**：任一子 Agent 执行失败 → 该组降级由主 Agent 自行检查 → 输出中标注 `⚠️ 自动降级`。

**汇总输出格式**：

```
📋 模块审核: <模块名>  (并行: 4 组 × 子 Agent)

□ 文件头注释 ............. ✅ / ❌
□ 头文件保护名 ........... ✅ / ❌
□ 头文件独立性 ........... ✅ / ❌
□ 函数命名 ............... ✅ / ❌
□ 静态函数命名 ........... ✅ / ❌
□ 变量命名 ............... ✅ / ❌
□ 类型命名 ............... ✅ / ❌
□ 宏命名 ................. ✅ / ❌
□ 函数注释 ............... ✅ / ❌
□ 段标记命名 ............. ✅ / ❌   (模块名_层，不用 module)
□ drv 区段验证 ........... ✅ / ❌
□ 代码格式 ............... ✅ / ❌
□ Magic Number .......... ✅ / ❌
□ volatile .............. ✅ / ❌

结果: ✅ 通过（14/14）/ ⚠️ 需修改（X/14）/ ❌ 不通过
```

## /em mode add <模块名>

将审核通过的模块加入模块库：

```
前置条件: /em mode audit 必须返回 ✅ 通过

1. 检查模块文件是否存在
2. 再次验证 14 项规则（防篡改）
3. 复制 .c/.h 文件到 skills/em/modules/
4. 更新 modules/MODULES.md
```

```
✅ 已加入模块库: ela_queue.c/h
   等级: A（零 HAL 依赖，可直接复用）
```

```
❌ 拒绝加入: <模块名>
   原因: 14 项验证未全部通过
   请先修复以下问题:
   - [ ] 违规项1
   - [ ] 违规项2
```

## /em mode install <模块名>

从模块库复制到当前项目 `ELA_LIB/`：

1. 检查 `skills/em/modules/<模块名>.c/h` 是否存在
2. **递归解析依赖**：查 `modules/MODULES.md` 看本模块是否有依赖
   - 有依赖 → 对每个依赖模块**先执行 install**（同目录检查，已安装则跳过）
   - 依赖模块自身可能还有依赖 → 递归处理，直到无依赖为止
   - 安装顺序：依赖树自底向上（叶子节点先装）
3. **创建 ELA_LIB 目录结构**（首次安装时）：
   - 检查当前项目 `ELA_LIB/` 目录是否存在
   - 不存在则创建，并从模板生成 `elaco_main.c` 和 `elaco_main.h`
   - 模板路径：`plugins/embedded/templates/elaco_main_template.c/h`
   - 日期自动替换为当前日期（YYYY-MM-DD 格式）
   - 已存在 `elaco_main.c/h` → 跳过生成，保留用户自定义内容
4. 复制 .c/.h 文件
5. **更新 `elaco_main.h` 中的模块包含**（自动添加 `#include "ela_xxx.h"`）
   - 检查 `elaco_main.h` 中是否已包含该模块的头文件
   - 未包含 → 在 `/* 用户模块头文件将自动添加在此处 */` 注释后添加
   - 已包含 → 跳过
   - **确保添加的 `#include` 语句符合规范**（头文件包含顺序）
6. **编码规范检查**（14 项验证）
   - 检查安装的模块文件是否符合 `references/ela_rules.md` 规范
   - 不符合 → 输出违规项，阻止安装，提示用户修复
   - 符合 → 继续下一步
   - **规范检查是最高优先级**，不符合规范的代码禁止安装
7. 提示需要适配的内容（如有）

```
✅ 已安装: ela_pid.c/h → ELA_LIB/
   等级: A（零 HAL 依赖）
   依赖: ela_queue（已安装）
   📁 已创建 ELA_LIB 目录结构（含 elaco_main.c/h）
   ✅ 编码规范检查通过（14/14）
```

```
✅ 已安装: ela_queue.c/h → ELA_LIB/
   等级: A（零 HAL 依赖）
   本模块无依赖，可直接使用。
   ✅ 编码规范检查通过（14/14）
```

```
❌ 安装失败: <模块名>
   原因: 依赖模块 <xxx> 不在模块库中
           请先入库依赖模块，或手动处理。
```

```
❌ 安装失败: <模块名>
   原因: 编码规范检查未通过（X/14）
           违规项:
           - [ ] 违规项1
           - [ ] 违规项2
           请先修复规范问题，再安装。
```
