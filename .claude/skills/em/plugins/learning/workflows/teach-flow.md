# 教学工作流（基于 learn-everything）

> 整合 learn-everything 的核心教学能力，适配 em-skill 的项目管理场景。

## 核心理念

- 用用户的语言教学（默认中文）
- 优化持久理解，不只是总结
- 逐章推进，一次只教一个章节
- 练习答案在用户尝试前隐藏
- 维护学习状态（概念掌握表、复习队列）

## 输入模式

### 主题模式

用户提供主题名 → AI 调研并生成课程

1. 定义范围和目标能力
2. 识别前提条件
3. 使用 websearch/webfetch 收集权威资料
4. 区分稳定基础和当前发展
5. 陈述范围和假设

### 文档模式

用户提供 PDF/文档 → AI 读取并生成课程

1. 读取文档结构：目录、标题、章节长度
2. 短文档（≤30 页）：完整阅读
3. 长文档：从结构生成地图，按需阅读当前章节
4. 按语义单元划分（非等分页数）
5. 引用页码和章节名
6. 标记不可读/缺失/矛盾内容

### 续学模式

用户提供上次状态或说「继续」→ 恢复课程

1. 读取 course-state.md
2. 执行 2-4 个热身复习题（加权弱概念和错误类别）
3. 给出位置摘要
4. 进入当前章节继续

## 课程地图生成

**章节 sizing 规则**：
- 包含 >1 个主要推理跳跃 → 拆分
- 无法独立理解 → 合并
- 前提知识缺失 → 创建桥接章节
- 高级附录 → 可选（除非需要）
- 总章节数：3-8 个

**每个章节包含**：
- 简洁标题
- 1 个可观察学习目标
- 核心概念
- 前提链接
- 来源页码（文档模式）
- 1 个计划练习产出

## 教学循环

### 1. 定位

说明本章为什么重要，与之前学到的联系。

### 2. 目标

给出 2-4 个可观察目标。

### 3. 核心讲解

分小节讲解，每次引入一个概念跳跃。

### 4. 示例

演示推理过程，不只是最终结果。

### 5. 常见误区

识别 1-3 个可能的混淆或边界情况。

### 6. 主动回忆

在进入长练习前，问一个简短的检查问题。

### 7. 练习

根据当前掌握程度校准练习。

### 8. 反馈循环

评分学习者回答，决定：补救 / 提高难度 / 继续。

## 练习设计

**认知层次**（混合至少 2 个）：

| 层次 | 说明 | 示例 |
|------|------|------|
| 回忆 | 定义、列举、标记 | "DMA 传输有哪几种模式？" |
| 解释 | 用自己的话复述 | "用你的话解释什么是双缓冲" |
| 应用 | 解决代表性问题 | "配置 DMA 从 ADC 读取数据" |
| 调试 | 找出推理/代码中的错误 | "这段 DMA 配置有什么问题？" |
| 迁移 | 在新场景中应用 | "用 DMA 实现串口接收" |
| 综合 | 连接多个章节 | "设计一个 ADC+DMA+环形缓冲的采集系统" |

**默认**：每章 3-5 个练习，混合至少 2 个层次。

## 评分反馈

**正确时**：
- 逐步提高难度
- 如果可能猜对，要求解释
- 掌握强时给迁移题

**错误时**：
1. 先给提示（针对误区）
2. 再失败 → 给更强提示
3. 重复失败 → 揭示解答 + 紧跟近迁移题

## 概念掌握追踪

**状态定义**：

| 状态 | 说明 |
|------|------|
| unseen | 未接触 |
| introduced | 已讲解 |
| practicing | 正在练习 |
| mastered | 能独立解释和应用 |
| review-needed | 之前掌握但现在遗忘 |

**更新时机**：章节结束时 + 重大错误后。

## 复习调度

- 每章结束后：2-4 个累积复习题
- 课程约 1/3 和 2/3 时：混合检查点
- 课程结束：最终评估 + 综合任务

## 学习者命令

| 命令 | 动作 |
|------|------|
| 继续 / continue | 进入下一个待处理步骤 |
| 提示 / hint | 给下一个提示（不给完整答案） |
| 答案 / show solution | 揭示并解释答案，然后问近迁移题 |
| 更简单 / simpler | 降低抽象，增加前提和示例 |
| 更深入 / deeper | 增加推导、边界情况、更难迁移 |
| 跳过 / skip | 标记跳过，继续 |
| 复习 / review | 生成加权弱概念的检索练习 |
| 考试 / test me | 运行闭卷混合评估 |
| 进度 / progress | 显示已完成章节、掌握情况、下一步 |
| 保存 / save | 保存当前状态到文件 |

## 课程完成

1. 混合最终评估（先不显示答案）
2. 综合任务（使用多个章节）
3. 评分 + 概念级反馈
4. 简洁掌握报告
5. 个性化复习计划（聚焦弱概念）
6. 参考手册（评估后创建）

## 质量检查

每次响应前验证：
- 解答在用户尝试前未暴露
- 当前活动匹配课程状态
- 课程不假设未解释的前提
- 响应给出一个清晰的下一步动作
