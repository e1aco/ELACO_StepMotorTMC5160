# 知识库管理工作流

> 沉淀调试经验、技术要点、代码片段，支持检索复用。

## 核心理念

- 踩过的坑记下来，下次不踩
- 解决方案可检索，遇到类似问题快速定位
- 代码片段可复用，不重复造轮子

## 知识库目录结构

```
<STATE_DIR>/learning/kb/
├── debug/                   # 调试知识库
│   └── <slug>.md            # 问题 + 解决方案
├── concepts/                # 概念知识库
│   └── <slug>.md            # 技术要点
└── snippets/                # 代码片段
    └── <slug>.md            # 可复用代码
```

## 条目类型

### debug（调试经验）

记录遇到的问题和解决方案。

```markdown
# <问题标题>

**类型**: debug
**创建时间**: YYYY-MM-DD
**标签**: <tag1>, <tag2>

## 问题/背景
<遇到什么问题，现象是什么>

## 解决方案
<怎么解决的，步骤是什么>

## 根因分析
<为什么会出这个问题>

## 适用场景
<什么情况下会遇到这个问题>

## 预防措施
<如何避免再次发生>

## 相关链接
- 来源：<URL 或文档>
- 相关条目：<其他 slug>

## 变更记录
- YYYY-MM-DD: 初始创建
```

### concepts（技术要点）

记录技术概念和原理。

```markdown
# <概念名称>

**类型**: concept
**创建时间**: YYYY-MM-DD
**标签**: <tag1>, <tag2>

## 核心要点
<用 3-5 句话概括这个概念>

## 详细说明
<深入解释原理和机制>

## 使用场景
<什么时候用，怎么用>

## 常见误区
<容易搞错的地方>

## 参考资料
- <URL 或文档>

## 变更记录
- YYYY-MM-DD: 初始创建
```

### snippets（代码片段）

记录可复用的代码。

```markdown
# <片段名称>

**类型**: snippet
**创建时间**: YYYY-MM-DD
**标签**: <tag1>, <tag2>

## 用途
<这段代码做什么>

## 代码

\`\`\`c
// 代码内容
\`\`\`

## 使用方法
<怎么用这段代码>

## 参数说明
<关键参数的含义>

## 注意事项
<使用时要注意什么>

## 变更记录
- YYYY-MM-DD: 初始创建
```

## 添加流程

### 自动触发（/em result 后）

每次 `/em result` 执行后，追加询问：

```
💡 是否将此次经验沉淀到知识库？
   1. debug — 调试经验（问题+解决方案）
   2. concept — 技术要点（概念+原理）
   3. snippet — 代码片段（可复用代码）
   4. 跳过
```

用户选择后进入交互式添加。

### 手动添加（/em learn kb add）

```
📝 添加知识条目

类型: [debug] / concept / snippet
> debug

标题: <输入标题>

问题/背景:
> <输入问题描述>

解决方案:
> <输入解决方案>

标签（逗号分隔）: <输入标签>

✅ 已保存: .em/learning/kb/debug/<slug>.md
```

## 检索流程

### 搜索（/em learn kb search）

1. 遍历 `kb/` 下所有 `.md` 文件
2. 匹配文件名 + 内容中的关键词
3. 返回匹配条目的摘要 + 路径

### 检索优先级（项目开发时）

```
1. 知识库 debug/ — 之前踩过的坑
2. 知识库 concepts/ — 技术要点
3. 知识库 snippets/ — 可复用代码
4. 项目 Require/ 文档
5. 网上搜索（需用户确认）
```

## 维护规则

| 规则 | 说明 |
|------|------|
| slug 命名 | kebab-case，简洁明了（如 `can-bus-timeout`） |
| 标签 | 2-5 个，帮助分类和检索 |
| 去重 | 添加前搜索是否已有类似条目 |
| 更新 | 问题有新解决方案时更新原条目，不新建 |
| 归档 | 过时条目标记 `status: archived`，不删除 |

## 与其他模块的集成

| 模块 | 集成方式 |
|------|----------|
| `/em result` | 结束后询问是否沉淀 |
| `/em verify` | 失败时建议沉淀 |
| `/em new` | 开发前检索是否有类似经验 |
| `/em rec` | 恢复时显示知识库统计 |
