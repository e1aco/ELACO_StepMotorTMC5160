# 命令: /em learn teach <topic> [source]

## 功能

交互式教学：基于主题或文档，生成结构化课程并逐章教学。整合 learn-everything 的核心教学能力。

## 触发

```
/em learn teach <topic>                # 主题模式：AI 调研并生成课程
/em learn teach <topic> --doc <path>   # 文档模式：基于 PDF/文档生成课程
/em learn teach --resume               # 续学模式：继续上次的课程
```

**示例**

```
/em learn teach CAN总线协议
/em learn teach STM32 DMA --doc ./refs/dma_guide.pdf
/em learn teach --resume
```

## 参数

| 参数 | 必填 | 说明 |
|------|------|------|
| topic | 是（续学时可省） | 主题名称或 slug |
| --doc <path> | 否 | 文档路径（PDF/MD/TXT） |
| --resume | 否 | 续学模式，继续上次课程 |

## 执行流程

### 步骤 1: 检查学习模式启用

读 `<STATE_DIR>/project.json`，确认 `type == "learning"` 或检测到 `<STATE_DIR>/learning/` 目录：

- 已启用 → 进入步骤 2
- 未启用 → 提示「学习模式未启用，是否启用？(Y/n)」，确认后写入 `project.json`

### 步骤 2: 确定输入模式

#### 续学模式（--resume）

1. 读 `<STATE_DIR>/learning/teach/` 下所有课程目录
2. 找到最后活跃的课程（最近修改的 course-state.md）
3. 读取 course-state.md，显示进度摘要
4. 执行 2-4 个热身复习题（加权弱概念）
5. 进入当前章节继续教学

#### 文档模式（--doc）

1. 读取文档结构：目录、标题、章节长度
2. 短文档（≤30 页）：完整阅读后生成课程地图
3. 长文档：从结构生成课程地图，按需阅读当前章节
4. 识别：目录、标题、前提条件、核心概念、定义、示例、图表
5. 按语义单元划分章节（非等分页数）

#### 主题模式（默认）

1. 定义合理的范围和目标能力
2. 识别前提条件，从基础到高级排列概念
3. 对于需要调研的主题，使用 websearch/webfetch 收集权威资料
4. 区分稳定基础和当前发展

### 步骤 3: 诊断用户水平

**简短诊断**（不拖延进度）：

```
📚 开始学习: <topic>

在开始之前，快速了解一下你的水平：
1. 你对这个主题了解多少？（新手/有基础/熟练）
2. 你想达到什么目标？（理解概念/能用/能教别人）

直接回答即可，或者说「跳过」开始学习。
```

### 步骤 4: 生成课程地图

**格式**：

```markdown
# 学习计划: <topic>

**目标能力**: <学完后能做什么>
**起始水平**: <诊断结果>
**来源**: <文档路径或"主题调研">

| 章节 | 目标 | 核心概念 | 来源 | 练习产出 |
|------|------|----------|------|----------|
| 1. ... | ... | ... | pp. ... | ... |
| 2. ... | ... | ... | ... | ... |

**学习方式**: 逐章学习，先尝试练习再看答案。
```

**章节 sizing 规则**：
- 包含 >1 个主要推理跳跃时拆分
- 无法独立理解的章节合并
- 前提知识缺失时创建桥接章节
- 总章节数 3-8 个

### 步骤 5: 保存课程状态

创建目录和文件：

```
<STATE_DIR>/learning/teach/<slug>/
├── course-state.md    # 课程状态（概念表、进度）
├── course-map.md      # 课程地图
└── notes.md           # 学习笔记（可选）
```

**course-state.md 格式**：

```markdown
# 课程状态: <topic>
**学习者**: <水平, 目标, 节奏 — 一行>
**来源**: <文件/主题 + 范围>

## 章节
- [x] 1. [标题] — 已掌握: [概念] | 弱: [概念]
- [ ] 2. [标题]  ← 当前, [章节内位置]
- [ ] 3. ...

## 概念表
| 概念 | 状态 | 证据 | 下次复习 |
|------|------|------|----------|

## 复习队列
- [概念] — 上次: [章节 N] — [结果]

## 错误日志
- [章节 N]: [主要错误类型, 一行]
```

更新 `_index.json` 追加主题元数据。

### 步骤 6: 开始教学

进入第一个章节，执行教学循环。

## 教学循环

每个章节按以下循环：

### 1. 定位

说明本章为什么重要，与之前学到的有什么联系。

### 2. 目标

给出 2-4 个可观察的学习目标。

### 3. 核心讲解

分小节讲解，每次引入一个概念跳跃。

### 4. 示例

演示推理过程，不只是最终结果。

### 5. 常见误区

识别 1-3 个可能的混淆或边界情况。

### 6. 主动回忆

在进入长练习前，问一个简短的检查问题。

### 7. 练习

根据当前掌握程度校准练习。

### 8. 反馈循环

评分学习者回答，决定：补救 / 提高难度 / 继续。

## 练习设计

**认知层次**（混合至少 2 个）：

| 层次 | 说明 |
|------|------|
| 回忆 | 定义、列举、标记 |
| 解释 | 用自己的话复述；比较概念 |
| 应用 | 解决代表性问题 |
| 调试 | 找出推理/代码中的错误 |
| 迁移 | 在新场景中应用 |
| 综合 | 连接多个章节或产出 |

**默认**：每章 3-5 个练习，混合至少 2 个层次。

## 评分反馈

**格式**：

```
**判定**: [正确 / 大部分正确 / 部分正确 / 需要修改]

**做得好的**: ...
**关键差距**: ...
**纠正**: ...

**下一步**: [修改要求 / 跟进问题 / 继续]
```

**错误处理**：
1. 先给提示（针对误区）
2. 再失败 → 给更强提示
3. 重复失败 / 明确要求 → 揭示解答 + 紧跟近迁移题

## 概念掌握追踪

在 course-state.md 中维护概念表：

| 状态 | 说明 |
|------|------|
| unseen | 未接触 |
| introduced | 已讲解 |
| practicing | 正在练习 |
| mastered | 能独立解释和应用 |
| review-needed | 之前掌握但现在遗忘 |

**更新时机**：章节结束时 + 重大错误后。

## 复习调度

- 每章结束后：2-4 个累积复习题
- 课程约 1/3 和 2/3 时：混合检查点
- 课程结束：最终评估 + 综合任务

## 学习者命令

识别自然命令：

| 命令 | 动作 |
|------|------|
| 继续 / continue | 进入下一个待处理步骤 |
| 提示 / hint | 给下一个提示（不给完整答案） |
| 答案 / show solution | 揭示并解释答案，然后问近迁移题 |
| 更简单 / simpler | 降低抽象，增加前提和示例 |
| 更深入 / deeper | 增加推导、边界情况、更难迁移 |
| 跳过 / skip | 标记跳过，继续 |
| 复习 / review | 生成加权弱概念的检索练习 |
| 考试 / test me | 运行闭卷混合评估 |
| 进度 / progress | 显示已完成章节、掌握情况、下一步 |
| 保存 / save | 保存当前状态到文件 |

## 课程完成

课程结束时提供：

1. 混合最终评估（先不显示答案）
2. 综合任务（使用多个章节）
3. 评分 + 概念级反馈
4. 简洁掌握报告
5. 个性化复习计划（聚焦弱概念）
6. 参考手册（评估后创建，不替代检索练习）

## 与通用核的接口

- `/em learn teach` 创建的子工程可在 `topics/<slug>/code/` 内独立运行 `/em new`
- 学习状态与项目开发状态并行不冲突
