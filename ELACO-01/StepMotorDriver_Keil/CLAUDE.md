## Project Overview

NBYT-004 dual-axis step motor driver. MCU: **STM32F407ZET6** (Cortex-M4, 168 MHz). Two **TMC5160** driver chips (U1, U2) on a shared SPI3 bus with individual chip selects. Toolchain: **Keil MDK-ARM**, project generated by **STM32CubeMX**.

- CubeMX project file: `StepMotorDriver_Keil.ioc`
- Keil project: `MDK-ARM/StepMotorDriver_Keil.uvprojx`
- Preprocessor defines: `USE_HAL_DRIVER;STM32F407xx`

## How to Build

Open `MDK-ARM/StepMotorDriver_Keil.uvprojx` in Keil MDK-ARM and build (F7). Alternatively, regenerate code from `StepMotorDriver_Keil.ioc` in CubeMX, then build in Keil.

No CLI build is configured; there is no Makefile or CMake.

## Architecture: Three-Layer Structure

### Layer 1: Drivers (vendor, auto-generated)
`Drivers/CMSIS/` and `Drivers/STM32F4xx_HAL_Driver/` — STM32 HAL and CMSIS. Treat as read-only; CubeMX regenerates these when the `.ioc` is modified.

Enabled HAL modules (`Core/Inc/stm32f4xx_hal_conf.h`): CAN, IWDG, SPI, TIM, UART, GPIO, EXTI, DMA, RCC, FLASH, PWR, CORTEX.

### Layer 2: Core (CubeMX-generated, user-editable via USER CODE blocks)
`Core/Src/` and `Core/Inc/` — Peripheral init (`MX_*_Init()`), ISR handlers (`stm32f4xx_it.c`), MSP init (`stm32f4xx_hal_msp.c`), `main.c`, `system_stm32f4xx.c`. Preserve `/* USER CODE BEGIN ... */` / `/* USER CODE END ... */` markers — CubeMX uses them to preserve custom code across regeneration.

Peripherals initialized in `main()`:
- **SPI3** (PC10=SCK, PC11=MISO, PC12=MOSI) — Mode 3 (CPOL=HIGH, CPHA=2EDGE), master, 8-bit, NSS software, prescaler /2 → ~21 MHz
- **CAN1** — for external device communication
- **USART1** — DMA-based variable-length RX with IDLE line detection; 256-byte DMA circular buffer (`g_rx1_buf`, `g_rx1_offset`). IDLE ISR is handled in `USER_UART_IRQHandler()` (called from `USART1_IRQHandler` **before** `HAL_UART_IRQHandler`)
- **TIM2, TIM3, TIM4, TIM5** — TIM4_CH1 outputs PWM; TIM5 generates 5ms periodic interrupt for button scanning
- **IWDG** — independent watchdog, refreshed in main loop

Clock: HSE (8 MHz) → PLL (PLLM=4, PLLN=84, PLLP=2) → 168 MHz SYSCLK; APB1 = 42 MHz, APB2 = 84 MHz.

### Layer 3: ELA_LIB (custom middleware — the core application logic)
`ELA_LIB/` contains all product-specific code. Key modules:

| Module | Purpose |
|--------|---------|
| `elaco_main.c/h` | Main loop (`elaco_main()`), all HAL callback implementations (UART Rx Half/Full/IDLE, TIM period elapsed, CAN Rx), EEPROM-backed LED state persistence |
| `ela_tmc5160.c/h` | TMC5160 SPI register-level driver. `WriteReg()` sends 40-bit frame (write: `0x80|addr + 4B data`), `ReadReg()` needs two SPI transactions per TMC5160 datasheet §4.1.1. GPIO macros for SD_MODE, SPI_MODE, CS, DRV_ENN per chip. Full register map defined. |
| `ela_eeprom.c/h` | Internal Flash EEPROM emulation. Sector 4 (64KB at `0x08010000`) for calibration, Sector 5 (128KB at `0x08020000`) for data. Write uses halfword Flash programming. `Eeprom_Valid()` checks if first word ≠ `0xFFFFFFFF`. |
| `ela_button.c/h` | MultiButton state machine (embedded, not separate lib). SW1 = PD1, active low. 10ms debounce, short/long/double click detection. `Button_ScanProc()` called every 5ms from TIM5 ISR. `OnPress()` and `OnLongPress()` are `__weak` — override in `elaco_main.c`. |
| `ela_can.c/h` | CAN init and send helpers. Supports 11-bit (`Can_SendMessage`) and 29-bit (`Can_SendMessageExt`) IDs. |
| `ela_uart.c/h` | UART DMA init with 256B circular buffer for variable-length RX. |
| `ela_timer.c/h` | Timer init for periodic tasks. |
| `ela_queue.c/h` | 48-element × 8-byte CAN command queue (`QUEUE_T`), used with `g_can_queue_st`. |
| `FreeModbus/` | FreeModbus protocol stack — ASCII and RTU enabled, TCP disabled (`mbconfig.h`). Port layer in `port.c/h/serial/timer/event.c`. Currently integrated but not yet activated in main loop. |
| `tmc5160_example.c` | TMC5160 usage cookbook — init, chopper config, position move, velocity mode, StallGuard2 homing. Reference when implementing TMC5160 features. |

### Pin Definitions (from `main.h`)

**TMC5160 U1**: CS=`U1_SPI1_SCN` (PD5), DRV_ENN (PG13), SWN_DIAG0 (PG11), SWP_DIAG1 (PG12), REFL_DIR (PD4), SD_MODE (PD3), SPI_MODE (PD2)
**TMC5160 U2**: CS=`U2_SPI1_SCN` (PB0), DRV_ENN (PB1), SWN_DIAG0 (PF12), SWP_DIAG1 (PF11), REFL_DIR (PF13), SD_MODE (PB12), SPI_MODE (PB13)
**Other**: MCU_LED (PA1), TEST_LED (PA2), SW1 button (PD1), TMC_CLK (PB6)

### Key Design Patterns

- **Weak function overrides**: `ela_button.c` defines `__weak OnPress()` and `__weak OnLongPress()`. `elaco_main.c` provides the strong definitions.
- **EEPROM delayed write**: Button press toggles LED state in RAM immediately; Flash write is deferred 500ms to debounce and reduce erase cycles.
- **UART IDLE detection**: `USER_UART_IRQHandler()` must be called **before** `HAL_UART_IRQHandler()` in the USART1 ISR — otherwise HAL may clear the IDLE flag.
- **TMC5160 SPI access pattern**: Read requires 2 back-to-back `HAL_SPI_TransmitReceive()` calls with CS held low between them (but this implementation toggles CS between the two calls).
