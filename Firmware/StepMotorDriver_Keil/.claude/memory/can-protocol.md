## 接收命令协议 (CAN ID: `0x1AA55F42`)

| 字节索引 | 含义 | 说明 |
|:----|:----------|:-------------------------------------------------------------|
| [0-3] | 位置/速度值 (int32) | 位置模式=目标位置, 速度模式=目标速度, 停止命令=0。4字节小端 |
| [4] | 命令码 | 0x01=绝对位置移动, 0x02=相对位置移动, 0x03=速度模式, 0x04=停止/刹车 |
| [5] | 电机选择 | 0x01=电机1(U1), 0x02=电机2(U2), 0x06=两个电机 |
| [6] | 运动参数组 | 0x01=低速精准(VMAX=5k,AMAX=1k), 0x02=中速(VMAX=20k,AMAX=5k), 0x03=快速(VMAX=50k,AMAX=10k), 0x04=高速(VMAX=100k,AMAX=20k) |
| [7] | 校验和 | 前7字节累加取低8位 |

**命令码：**
- `0x01` 绝对位置移动 — 运动到字节[0-3]的绝对位置
- `0x02` 相对位置移动 — 从当前位置偏移字节[0-3]（正=正转, 负=反转）
- `0x03` 速度模式 — 字节[0-3]为目标速度，持续旋转直到收到停止命令
- `0x04` 停止/刹车 — 立即停止，保持锁轴

## 反馈响应协议 (CAN ID: `0x1AA55F43`)

| 字节索引 | 含义 | 说明 |
|:----|:-------------|:---------------------------------------------------------- |
| [0-3] | 位置值 (int32) | 当前XACTUAL绝对位置值（两个电机时发送0） |
| [4] | 状态标志位 | bit0=到位, bit1=堵转/失步, bit2=过温, bit3=驱动错误 |
| [5] | 电机选择 | 0x01=电机1(U1), 0x02=电机2(U2), 0x06=两个电机 |
| [6] | 运动阶段 | bit0=加速中, bit1=匀速, bit2=减速中, bit3=归零等待, bit4=静止锁轴 |
| [7] | 校验和 | 前7字节累加取低8位 |

**字节[4] 状态标志位：**
```
bit0 → 0=运动中，1=已到位
bit1 → 0=正常，1=堵转/失步
bit2 → 0=正常，1=过温告警
bit3 → 0=正常，1=驱动错误
```
示例: `0x00`=运动中正常, `0x01`=已到位正常, `0x03`=已到位+堵转, `0x05`=已到位+过温

## 硬件
- MCU: **STM32F407ZET6**
- Driver: TMC5160 via SPI3, 芯片1=电机1(U1), 芯片2=电机2(U2)
- CAN: CAN1, 扩展ID 29bit
- 当前板子2个电机, 0x06表示两个电机
