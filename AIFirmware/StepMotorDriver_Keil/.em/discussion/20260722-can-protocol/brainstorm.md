# brainstorm: S1-can-protocol

## 需求理解

实现 CAN 通信模块，完成：
1. **接收**：CAN ID `0x1AA55F42`，解析 6 种命令码（绝对/相对位置、速度、停止、PID调参），含校验和验证
2. **发送**：CAN ID `0x1AA55F43`，运动反馈（位置+状态）+ 调参反馈
3. **缓冲**：中断接收 → `ela_queue` 入队 → 主循环取出处理
4. **HAL对接**：在 CubeMX 已初始化的 CAN1 基础上，增加过滤器配置、中断接收、发送函数

## 选定方案

**方案 B**: 分层 ela_can_drv + ela_can_usr
- drv 层：HAL 薄封装（过滤器、中断、发送）
- usr 层：协议解析 + 队列管理 + 反馈组装

## 关键决策

- 2026-07-22: CAN 模块采用 drv/usr 分层架构
