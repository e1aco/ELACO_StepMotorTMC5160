# 会话日志: 编码器调试

**日期**: 2026-07-23
**会话类型**: 调试

## 调试内容

### 1. 编码器方向问题
- 现象：XACTUAL=51200, X_ENC=-51230（反向且有偏差）
- 根因：ENC_CONST 符号决定编码器计数方向，不是 ENCMODE 的 pol_A/pol_B
- 解决：取反 ENC_CONST (0x000CCCCD → 0xFFF33333)

### 2. ENCMODE 误解
- ENCMODE 的 pol_A/pol_B 只影响 N 通道事件检测，不影响计数方向
- 改 ENCMODE=0x01 导致电机不转（影响了其他功能）

### 3. 电机电流调整
- IHOLD: 6 → 4 (保持电流 35mA → 26mA)
- IRUN: 16 → 10 (运行电流 105mA → 65mA)

### 4. CAN 反馈格式
- 最终：byte[0-3] 发送偏差值 (X_ENC - XACTUAL)
- byte[4]: 状态标志位

## 调试结果

| 项目 | 结果 |
|------|------|
| 编码器方向 | ✅ 取反 ENC_CONST 解决 |
| 编码器偏差 | ✅ 在 256 以内 |
| 电机电流 | ✅ 降低后发热改善 |
| CAN 反馈 | ✅ 偏差值正确显示 |

## 关键知识点

1. TMC5160 ENC_CONST 是 16.16 定点数，符号位控制计数方向
2. ENCMODE 的 pol_A/pol_B 只影响 N 通道事件检测
3. 增量编码器上电没有机械零点，会有初始偏移
4. ENC_DEVIATION 阈值应大于初始偏移量（256 足够）
