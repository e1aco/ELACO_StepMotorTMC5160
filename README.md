## 设置CAN协议
### 1. 接收命令协议 (CAN ID: `0x1AA55F42`)

| 字节索引  | 含义             | 说明                                                           |
| :---- | :------------- | :----------------------------------------------------------- |
| [0-3] | 位置/速度值 (int32) | 位置模式=目标位置, 速度模式=目标速度, 停止命令=0。4字节小端（低字节在前）                    |
| [4]   | 命令码            | 0x01=绝对位置移动, 0x02=相对位置正转, 0x03=相对位置反转, 0x04=速度模式, 0x05=停止/刹车 |
| [5]   | 电机选择           | 0x01=U1, 0x02=U2, 0x03=U3, 0x04=U4, 0x05=U5, 0x06=所有电机       |
| [6]   | 运动参数组          | 选择预配置的速度+加速度组合(0x01~0x04)，见下方参数组定义                           |
| [7]   | 校验和            | 前7字节累加取低8位                                                   |

**命令码说明：**
- `0x01`（绝对位置移动）：电机以指定参数组运动到字节[0-3]的绝对位置
- `0x02`（相对位置正转）：电机以指定参数组从当前位置正向移动字节[0-3]的步数
- `0x03`（相对位置反转）：电机以指定参数组从当前位置反向移动字节[0-3]的步数
- `0x04`（速度模式）：字节[0-3]为目标速度(int32)，电机持续旋转直到收到停止命令
- `0x05`（停止/刹车）：立即停止当前运动，保持锁轴状态

**运动参数组定义（代码中预配置）：**

| 参数组  | 说明   | VMAX    | AMAX   | 典型场景  |
| :--: | :--- | :------ | :----- | ----- |
| 0x01 | 低速精准 | 5,000   | 1,000  | 精准定位  |
| 0x02 | 中速普通 | 20,000  | 5,000  | 日常运动  |
| 0x03 | 快速   | 50,000  | 10,000 | 快速移动  |
| 0x04 | 高速   | 100,000 | 20,000 | 长距离搬运 |

### 2. 反馈响应协议 (CAN ID: `0x1AA55F43`)
| 字节索引  | 含义          | 说明                                                                                              |
| :---- | :---------- | :---------------------------------------------------------------------------------------------- |
| [0-3] | 位置值 (int32) | 当前电机的XACTUAL绝对位置值                                                                               |
| [4]   | 状态标志位       | 每个bit独立表示一个状态：bit0=到位, bit1=失步, bit2=过温, bit3=驱动错误, bit4=SPI通讯异常, bit5-7保留 （堵转先不看）              |
| [5]   | 电机选择        | 0x01=U1, 0x02=U2, 0x03=U3, 0x04=U4, 0x05=U5, 0x06=所有电机                                          |
| [6]   | 运动阶段        | 感觉只需要知道电机在运动或者停止就好了bit0=1运动 =0停止  //bit0=加速中, bit1=匀速, bit2=减速中, bit3=归零等待, bit4=静止锁轴, bit5-7保留 |
| [7]   | 校验和         | 前7字节累加取低8位                                                                                      |

**字节[4] 状态标志位说明：**
```
bit0 → 0=运动中，   1=已到位
bit1 → 0=正常，     1=堵转/失步
bit2 → 0=正常，     1=过温告警
bit3 → 0=正常，     1=驱动错误
bit4 → 0=SPI正常，  1=SPI通讯异常（读寄存器返回全F）
bit5-7 → 保留，固定为0
```
每个 bit 互不影响，可以同时为 1。例如：
- `0x01` = bit0=1，已到位，无异常
- `0x03` = bit0=1 + bit1=1，已到位，但检测到堵转
- `0x11` = bit0=1 + bit4=1，已到位，但存在SPI通讯异常
- `0x00` = 运动中，一切正常

**字节[6] 运动阶段说明：**
```
bit0 → 加速中
bit1 → 匀速
bit2 → 减速中
bit3 → 归零等待
bit4 → 静止锁轴
bit5-7 → 保留
```
- `0x01`（绝对位置移动）和 `0x02`/`0x03`（正/反转）：**立即发送一次**启动反馈（status=0x00 运动中），电机**到位后再发一次**到位反馈（status=0x01 已到位）
- `0x04`（速度模式）和 `0x05`（停止/刹车）：仅立即发送一次
- **异常反馈**：堵转/失步(bit1)、过温(bit2)、驱动错误(bit3)、SPI通讯异常(bit4)任一状态发生变化时，由5ms定时器触发立即发送

# 理论部分
## 1.关于原理图设计
1. **一般我们会设计VSA与VS电源由一路供电,这样是最安全的**，但是如果VS的电源超过了24V就不建议一路供电了，这时VSA应该使用单独的开关电源转换过来的电压进行供电，并且要保证在电机运行期间VSA不会跌落，正常如果要断VSA的电前应该先关断电机驱动(DRV_ENN脚拉高)。
2. 如果电机的输入电压允许12V的话，或者电机运行电压非常大的话，建议**使用DCDC转换为12V给VSA和12VOUT供电**（这时12VOUT内部的LDO可以直接不工作，进而会减少功率的损耗以及发热问题）
3. MOS管的选型上
	1. 将MOSFET调整到所需的电机电压(在峰值电源电压上增加5-10V的余量)和所需的最大电流，使得电阻功耗对于所选MOSFET封装的热容量仍然很低。
	2. 管子Vth选择10V内的（主要因为芯片驱动电压在10V左右）
	3. 对于选型好mos以后栅极电阻的选择，根据米勒电荷对阻值进行选择（同时还要考虑软件上的驱动电流设置DRVSTRENGTH）
	![alt text](mos1.png)
	4. 对于驱动电阻，如果驱动电阻的电阻超过表中给出的值时可以增加一个反向二极管（负极对驱动芯片）以增加开关速度。
	那么根据以上条件，我的mos选择AOD4126的米勒电荷是10nC(VGS=10V, VDS=50V, ID=20A),那么我的栅极电阻选择就是10R左右，驱动电流设置就是0/1。（此外我还加入了4148，以求更快的开关速度）
4. 为避免驱动器IC在其BM引脚到地会出现低于-5V的峰值，建议添加一个TVS二极管保护。
## 2.关于PCB布线
1. 将采样电阻和所有滤波电容尽可能靠近功率MOSFET。TMC5160靠近MOSFET放置，短线互连，以最小化寄生电感。
2. 将所有的GND、GNDA、GNDD及采样电阻GND连成一个公共地。
3. 5VOUT滤波电容直接连到5VOUT和GNDA引脚。
4. 需要做好对于芯片TMC5160的散热处理，特别是在MOSFET栅极电荷高、斩波器频率高或系统时钟频率大于12MHz的情况下，内部的LDO功耗较大。
5. 做好芯片的散热，使用内部LDO，损耗功率蛮高的。
6. 调整电桥布局，使环路电感最小。布局最好紧凑。
7. MOSFET 栅极连接短而直，走线较宽。
8. 输出电容采用470PF至几个NF。（103 102）

### SPI通信协议
- **数据帧**: 40位(5字节)，由CS片选信号定界
- **写命令**: Byte0 bit7=1 + 7位地址 + Byte1-4为32位数据
- **读命令**: Byte0 bit7=0 + 7位地址，需要两报通信
  - 第一报: 发送读命令，接收上一次通信数据(丢弃)
  - 第二报: 再次发送读命令，接收本次请求的寄存器数据
- **SPI模式**: CPOL=1, CPHA=1 (模式3)
